# Mobile Shell Safe Area Design

**项目**: `memo-inbox-client`
**日期**: `2026-04-15`
**范围**: `packages/app-core` 移动端壳层与收件流页面的布局修复
**状态**: 已确认设计，待用户审阅

## 1. 背景

当前移动端已经具备：

- 顶部状态栏避让
- 固定底部导航
- 收件流列表、标签、图片和编辑入口

但在手机端存在两个直接影响使用的问题：

- 部分 memo 内容在极端长度下会把右侧内容列撑宽，出现左右滑动
- 底部导航虽然可用，但当前安全区与手势区的处理较粗糙，底部视觉衔接不自然

这两个问题都发生在现有移动端布局模型内部，不需要借机重做整套移动端信息架构。

## 2. 目标

本次只解决以下目标：

- 手机端页面保持单屏宽度，不出现横向滚动
- 长文本、长链接、连续字符在窄屏下自然断行
- 底部导航与系统手势区的衔接更稳定、更自然
- 顶部现有状态栏适配不回退

## 3. 非目标

本次明确不包含：

- 移动端信息结构重构
- 顶部菜单交互重做
- 底部导航入口增删
- 收件流视觉语言重设计
- 其它移动页面统一改版
- 服务端接口或 DTO 调整

## 4. 方案选择

### 方案 A：最小补丁

不采用。

做法：

- 只给正文补断行规则
- 只给外层补 `overflow-x-hidden`
- 只给底栏补一点安全区内边距

问题：

- 只能掩盖当前症状
- 布局责任仍然分散在壳层和页面里
- 后续其它移动页面仍容易重复出现类似问题

### 方案 B：壳层收口 + 页面局部修复

这是本次采用方案。

做法：

- 由 `MobileShell` 统一控制移动端一屏高度、主滚动区和底部安全区
- 由 `MobileInbox` 修正内容列收缩、文本断行和局部容器宽度约束

优点：

- 改动范围可控
- 同时解决横向溢出和底部安全区问题
- 不需要改变现有页面结构和产品能力表达

### 方案 C：整体重做移动端壳层

不采用。

原因：

- 超出当前问题本身
- 需要重新验证更多页面
- 风险与收益不成比例

## 5. 布局设计

### 5.1 壳层职责

`MobileShell` 负责以下统一行为：

- 外层使用稳定的一屏高度容器
- 控制主滚动区只纵向滚动
- 统一管理底部导航占位与安全区
- 作为移动端横向溢出的最后一道兜底边界

壳层之外的页面不再通过固定魔法数字自行猜测底栏高度。

### 5.2 主内容区

主内容区保持当前纵向滚动模式，但调整为：

- 明确只允许 `overflow-y-auto`
- 禁止横向滚动
- 底部预留与底部导航真实视觉高度一致

目标是：

- 最后一条 memo 不被底部导航遮挡
- 页面底部不出现明显多余空白

### 5.3 底部导航

底部导航继续固定在底部，但处理方式改为：

- 导航本体包含 `env(safe-area-inset-bottom)` 的安全区内边距
- 可点击内容与安全区形成一个连续视觉容器
- 高度不再仅依赖单一固定值硬撑

目标不是隐藏系统手势条，而是让导航与底部系统区域的视觉关系自然。

## 6. 页面内容修复

### 6.1 Memo 列表项

`MobileInbox` 中每条 memo 的右侧内容列需要补齐可收缩约束：

- 右侧主内容列增加 `min-w-0`
- 相关子容器避免因默认最小宽度而拒绝收缩

这是本次修复横向溢出的第一原因位。

### 6.2 正文断行

正文继续保留现有 `whitespace-pre-wrap`，同时增加：

- 长单词/长链接可断行
- 连续英文数字可断行
- 极端无空格文本可在窄屏内折行

要求是：

- 保留原本的换行语义
- 不把正文裁剪成省略号
- 不引入横向滚动作为展示方式

### 6.3 附件与标签区域

保持当前结构，但补齐宽度约束：

- 图片网格最大宽度不能超过内容列可用宽度
- 标签区继续允许换行
- 删除确认按钮区在极端文案下也不能把整行撑宽

本次不改变它们的交互形式，只修正布局行为。

## 7. 技术落点

### 7.1 壳层

修改位置：

- `packages/app-core/src/screens/MobileShell.tsx`

变更内容：

- 收口移动端一屏容器与滚动区职责
- 调整主内容区底部预留逻辑
- 调整底部导航安全区处理
- 增加横向溢出保护

### 7.2 收件页

修改位置：

- `packages/app-core/src/screens/MobileInbox.tsx`

变更内容：

- 为 memo 列表右侧内容列增加可收缩约束
- 为正文增加断行能力
- 为图片、标签和操作区补齐宽度与换行约束

### 7.3 样式范围

本次优先在组件类名内完成修复。

不计划：

- 新增全局视觉主题
- 改动共享设计 token
- 为单一问题扩散出新的样式层级

## 8. 测试与验证

本次按最小但有效的方式验证：

1. 为 `MobileInbox` 增加失败测试，覆盖超长内容时的收缩与断行约束存在。
2. 为 `MobileShell` 增加失败测试，覆盖主滚动区与底部导航安全区处理。
3. 运行受影响包的 `test`。
4. 运行受影响包的 `typecheck`。

如果环境允许，再补一次移动端本地预览检查，但不以人工目测替代自动验证。

## 9. 最小验收标准

1. 手机端收件页默认不出现横向滚动。
2. 超长 memo 文本、长链接、连续字符会在窄屏内断行。
3. 最后一条内容不会被底部导航遮挡。
4. 底部导航与系统手势区的视觉衔接比当前稳定。
5. 顶部状态栏适配不回退。

## 10. 风险与约束说明

- 本次只修正移动端布局行为，不承诺解决所有页面的统一壳层历史问题。
- `overflow-x-hidden` 只能作为最后兜底，不能替代内容列真实收缩修复。
- 底部短横线来自系统手势区域，本次目标是改善与导航的关系，不是控制系统 UI 本身。
